# Оптимизация и синхронизация портфеля

Статичная HTML-презентация концепции новой фичи AI-Скринера. Не требует сборщика и внешних библиотек.

## Локальный запуск

Самый простой вариант — открыть `index.html` в браузере. Для запуска через локальный сервер:

```bash
python3 -m http.server 8000
```

После этого откройте `http://localhost:8000`.

## Production build

Сборка не требуется: `index.html`, `styles.css` и `main.js` уже являются production-ready статическими файлами. Если хостинг ожидает output directory, укажите корень проекта (`.`).

## Публикация и shareable link

- **GitHub Pages:** загрузите файлы в репозиторий, откройте Settings → Pages, выберите `Deploy from a branch` и корневую папку ветки. После публикации GitHub покажет публичную ссылку.
- **Vercel:** подключите репозиторий, оставьте Framework Preset = `Other`, Build Command пустым, Output Directory = `.`.
- **Netlify:** подключите репозиторий, оставьте Build Command пустым, Publish Directory = `.`.

Если проект позднее будет переведён на Vite, стандартный сценарий публикации: `npm run build`, затем загрузить папку `dist` в GitHub Pages или указать `npm run build` и `dist` в настройках Vercel/Netlify.

## Где заменить demo data

- Показатели «Было / Станет» — секция `#comparison` в `index.html`.
- План сделок — таблица в секции `#plan`.
- Статусы синхронизации — секция `#sync`.
- Тексты CJM для реального и виртуального портфеля — атрибуты `data-real` и `data-virtual` в секции `#scenario`.

Все цифры в презентации демонстрационные и должны быть заменены после получения расчётного примера от аналитика.
